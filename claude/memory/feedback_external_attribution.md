Never add Claude attribution to any commit, PR title, or PR description — regardless of which repo it targets (zamp's own repos, forks, or external upstream repos).

Specifically, do NOT include:
- `Co-Authored-By: Claude ...` trailers in commit messages
- `🤖 Generated with [Claude Code](...)` footers
- Any "Generated by Claude" / "Co-authored by Claude" / 🤖 markers in PR bodies
- Mentions of Claude / Claude Code as the author or contributor

**Why:** User chose `includeCoAuthoredBy: false` in `~/.claude/settings.json` on 2026-05-03, generalizing an earlier rule that only covered external/upstream PRs. They want every commit and PR to read as their own work without exception. Original surface: declined Claude attribution on an upstream LiteLLM PR; later expanded to apply globally.

**How to apply:**
- The settings.json flag handles the `Co-Authored-By` trailer automatically.
- For PR descriptions and bodies, where attribution is added by hand or by skill templates (e.g. `pr-workflow`, default `gh pr create` body), explicitly omit any 🤖 / Claude / Claude Code marker.
- If a slash command, skill, or template tries to inject these, strip them before submitting.
- This is the global default. The only override is if the user explicitly asks for attribution on a specific commit/PR.
