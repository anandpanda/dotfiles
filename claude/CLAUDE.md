# Personal preferences for Claude Code

This file is shipped in the dotfiles repo as a reference. install.sh does not
deploy it. Apply manually on each machine where you want these preferences:

    cat ~/dotfiles/claude/CLAUDE.md >> ~/.claude/CLAUDE.md
    # then edit ~/.claude/CLAUDE.md to dedupe and refine

---

## Git push policy

Never run `git push` (any flavor — plain, `--force`, `-u`, branch pushes) or
any command/skill that pushes to remote without an explicit user instruction
or confirmation in the current turn. Same applies to `gh pr create` and
similar commands that implicitly push the local branch. **"Commit" ≠ "push".**

When a higher-level workflow (e.g. a `pr-workflow` skill, a "create PR"
request, or a chained commit-and-push request) would push, pause and confirm
first: state what you're about to push, what branch, what remote, and ask
before running it.

## Attribution policy

Never add Claude attribution to any commit, PR title, or PR description —
regardless of which repo it targets (own repos, forks, or external upstream).

Specifically, do NOT include:
- `Co-Authored-By: Claude ...` trailers in commit messages
- `🤖 Generated with [Claude Code](...)` footers
- Any "Generated by Claude" / "Co-authored by Claude" / 🤖 markers in PR bodies
- Mentions of Claude / Claude Code as the author or contributor

The `includeCoAuthoredBy: false` flag in settings.json handles the
auto-trailer. For PR descriptions and bodies (where attribution can be added
by hand or by skill templates), explicitly omit any 🤖 / Claude / Claude Code
marker.

If a slash command, skill, or template tries to inject attribution, strip
it before submitting.
